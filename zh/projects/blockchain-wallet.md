## 项目概述

传统区块链钱包依赖单一私钥签名交易。私钥一旦丢失或泄露，资产将无法找回。本项目实现了 3/5 门限钱包：至少需要 5 名管理员中的 3 人协同参与，才能完成任意一笔交易的签名。整个签名流程中，完整私钥从不在任何节点上出现。

## 系统架构

```
客户端  ->  API  ->  钱包核心  ->  签名层  ->  管理员节点 (x5)
                                                    |
                                             密码学原语
                                         (DKG, Shamir SS, VSS)
```

## 密码学设计

**分布式密钥生成 (DKG)：** 每位管理员生成随机多项式并向其他人发送份额。每位管理员的最终密钥份额是收到的所有份额之和，没有任何人能看到完整私钥。

**Shamir 秘密共享：** 使用 secp256k1 域上的二次多项式对秘密进行分割。任意 3 份通过拉格朗日插值即可恢复秘密；少于 3 份则不泄露任何信息。

**可验证秘密共享 (VSS)：** 管理员公布多项式系数承诺，使所有人能验证收到的份额是否合法，恶意管理员发出无效份额时会被立即发现。

**门限 ECDSA（三轮协议）：**
1. 每位签名者承诺一个随机数
2. 随机数公开并验证，计算联合随机数
3. 每位签名者生成部分签名，协调者合并输出完整签名

## 安全性分析

| 威胁场景 | 应对措施 |
|---|---|
| 管理员节点被攻破 | 单份额无法签名，至少还需 2 份 |
| 少于 3 名管理员合谋 | 数学上无法生成有效签名 |
| 中间人攻击 | 全组件之间使用 mTLS |
| 重放攻击 | 每次使用全新随机数 + 交易哈希中含 UUID |
| 审计日志篡改 | SHA-256 哈希链，任何修改都会破坏链完整性 |
| 密钥暴力破解 | 2^256 搜索空间；份额存储于 SGX 密封存储 |
